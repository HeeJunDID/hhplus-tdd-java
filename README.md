# 테스트 코드를 작성할 떄  유의점
## 유효성 검사는 Controller layer에 있어야하는가? vs Service Layer에 있어야 하는가?
→유효성 검사할 때 비즈니스쪽과 가까운지 유저와의 상호작용과 가까운지에 따라서 바뀌는 것이 맞다.

다만, 스프링 프레임워크 단에서 간단한 유효성 검사는 처리해주기 때문에 간단한 유효성 검사는 작성하지 않는 편이다.

## 테스트 코드를 얼마나 많이 작성해야 하는가?
→ 모든시간을 테스트 하는데 할애할 수는 없다. 기능구현하는데도 시간이 걸리고 테스트 코드를 작성하는데도 시간이 걸린다. 커버리지 올리는데 집중을 하다보면 주객이 전도될수도 있다.(소프트웨어 기능을 공고하게 만드는 것보다 커버리지 올리는데 급급) 중요한 것은 구현한 기능을 테스트할 수 있는 정도면 된다.

## 테스트코드명을 명확하게 하려다 보니 너무 장황해진다.
→ 테스트명이 명확한게 좋다. 테스트코드가 무엇을 테스트 하는지 알기 위해서다. 그렇기 때문에 간략하게 이해할 수 있는 정도면 충분하다.

## 하나의 케이스에서 여러 검증을 해야하는 경우에는 어떻게 해야하는가?
→ Junit에는 nested라는 기능이 있다. 여러개의 테스트를 묶어서 하나의 덩어리로 테스트 할 수 있다.
